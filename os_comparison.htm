Comparison of Operating Systems

Compared are these different operating systems:

  • Windows: Most popular desktop OS. High overall market share. Probably the best software and hardware compatibility of all OS, due to its dominant market position. Comes pre-installed on most PCs sold. The comparison below refers to Windows 11.
  • macOS: Unix. Significant market share and great software support. Comes pre-installed on all Apple Macs.
  • iOS: Unix. The mobile equivalent to macOS, pre-installed on the iPhone. Significant market share.
  • Android: Unix-like, based on Linux. Most popular smartphone OS, pre-installed on almost all smartphones that are not made by Apple. Highest market share of all end-user OS.
  • Chrome OS: Unix-like, based on Linux. Decent market share in certain markets (e.g. US). Preinstalled on Chromebooks.
  • GNU/Linux: Unix-like. Widespread use in the server and embedded markets, still significant as a desktop and smartphone OS. Most popular distributions for desktops include Ubuntu, Debian, OpenSUSE, Fedora, and Arch.
  • FreeBSD: Unix. Without any doubt the most popular of the BSD systems, is used in production or as part of their product by large companies (e.g. Apple, Sony, Netflix). Has desktop-specific distributions such as GhostBSD.
  • NetBSD: Unix. Has been part of various commercial products. General purpose OS that can be used on the desktop, i.e. it runs modern browsers and modern desktop environments.
  • OpenBSD: Unix. Fork of NetBSD. General purpose OS that can be used on the desktop. Has influenced other OS.
  • DragonFly BSD: Unix. Fork of FreeBSD. General purpose OS that can be used on the desktop. The least popular of the four BSD systems, but it has some interesting features such as the HAMMER2 file system.
  • Illumos: Unix. Open source fork of Solaris 10 that has several distributions ranging from commercial to hobbyist, and target various markets such as cloud, NAS, server, and desktop (e.g. OpenIndiana).
  • Haiku: One of the most promising of the alternative OS. Has a relatively complete selection of software, including a modern browser. Attempts to re-create BeOS, a commercially failed desktop OS of the late 90s/early 2000s.
  • ArcaOS: While it is a legacy OS catering towards remaining users of OS/2, with fundamental limitations (e.g. stuck to 32-bit), Arca Noae has undertaken many significant improvements such as hardware compatibility or the possibility of using more than 4 GB RAM. ArcaOS can also run modern browsers.
  • RISC OS: Legacy OS from the late 80s that was open sourced in 2018. Used to be popular as a desktop OS in the UK. Limited in its design (e.g. no pre-emptive multitasking, no support for multi-core CPUs or 64-bit). Unlikely to ever become "relevant" again besides its legacy user base. Yet, it has recently received a modern web browser (Iris) and was ported to the Raspberry Pi. The comparison below refers to RISC OS 5.
  • AmigaOS: Legacy OS from the 1980s with slow development progress. Back in the day it was an impressive OS for home users. Suffers from limited hardware support (32-bit PowerPC only) with no plans to change this. Disappointing browser situation (best browser is Odyssey Web Browser/OWB, which is still receiving updates as of 2022 but is fundamentally based on a 2019 version of Webkit with backports/updates added). Long-term outlook is unclear as it is proprietary, commercial OS. The comparison below refers to AmigaOS 4.
  • MorphOS: Despite its negligible market share and limited hardware compatibility (PowerPC only, mostly obsolete devices; no support for multiple CPU cores or >2 GB RAM), this appears to be a relatively polished OS and has the most modern web browser (Wayfarer) of all AmigaOS-like operating systems. Good enough to be a daily driver for some.
  • AROS: An open source reimplementation of AmigaOS 3.1. While suffering from relatively slow development and few developers, it might outlive AmigaOS and MorphOS in the long-term, as AROS runs on x86 hardware and has some other modern features the others lack (such as 64-bit support).
  • OpenVMS: An old operating system going back to the 1970s, which used to be sold on servers and workstations alike. Nowadays, it's only marketed towards the server market, in particular legacy OpenVMS users. Nevertheless, the DECWindows desktop environment is still being developed and officially supported. The current owner, VSI, recently ported OpenVMS to x86, although more with a focus on running as a virtual machine than on bare metal. OpenVMS has influenced Microsoft Windows.

Also worth mentioning...

There are of course many more operating systems, for an extensive list see here. I have not included the following:

  • OS that have never been marketed towards end users in any way, e.g. z/OS, IBM i, MCP, VSEn, VxWorks, TRON, NonStop OS, OS-9, BAE Systems STOP, ThreadX, FreeRTOS, ...
  • OS that are not developed anymore, e.g. OS/2, Mac OS 9, IRIX, MS-DOS, BeOS/ZETA, AtheOS/Syllable, SkyOS, Symbian, Plan 9, BlackBerry OS 7, BlackBerry 10, REX OS, Nokia Series 30, Nokia Series 40, Nokia Asha OS, ...
  • OS that have not had any updates since 2020 even though they are not confirmed to be dead, e.g. UnixWare, Minix, Inferno, PhantomOS, Ghost, ...
  • This still leaves an extensive list of OS that I didn't include but are worth mentioning:
    • Unix
      • AIX: Commercial Unix from IBM. Not marketed towards workstation/desktop use anymore (e.g. Firefox browser and CDE desktop haven't been updated in years). While it will be supported until at least 2033, the long-term outlook for this OS is unclear as IBM is more likely to focus on its competing offer Red Hat Enterprise Linux. AIX doesn't run on commonly available consumer hardware (only POWER ISA).
      • HP-UX: Commercial Unix from HP. Not marketed towards workstation/desktop use anymore (e.g. Firefox browser and CDE desktop haven't been updated in years). Doesn't run on commonly available consumer hardware (only Itanium ISA). End of life announced for 2025 for both Itanium hardware and HP-UX support. It's a dead end.
      • Solaris: Commercial Unix from Oracle. Would almost look the same as Illumos in this comparison (as Illumos is a fork of Solaris). While Solaris was historically used for workstations and still includes relatively up to date desktop software (e.g. modern GNOME desktop and Firefox browser), it is clear that Oracle is focusing on the commercial server market. Even though Solaris is still regularly getting new features and improvements, the long-term perspective has been put into doubt. The current version is still supported until at least 2037, which is generally seen as the end of life date for Solaris. Go on Oracle's website and you'll see that they're focused more on Oracle Linux rather than Solaris. The SPARC ISA, one of the main arguments to buy a Solaris computer, is on its way out as well.
    • Unix-like
      • 9front: While still actively developed, this is a fork of a legacy OS (Plan 9) that never found widespread adoption in the first place. Lacks basic applications expected by most users (such as a modern web browser). Hard to use for novice users due to its unusual interface which was influenced by Oberon.
      • 9ferno: A fork of Inferno, which itself is a descendant of Plan 9, but with a more "normal" user interface. It is a distributed operating system that can run either native (on bare metal) or hosted (as an application on top of another operating system).
      • QNX: Used to be a fine OS for the desktop, with mainstream browsers and a full desktop environment available. While it still has a modern built-in browser (using the Blink engine), the focus has shifted to embedded and automotive use. The Photon GUI has been completely removed, making it pointless as an end-user OS. BlackBerry 10 OS for smartphones was based on QNX but is also not developed anymore.
      • Redox: Relatively recent project (started 2015) and more of a hobbyist OS, mostly developed by Jeremy Soller. Currently still too immature to be used as a daily driver (e.g. no modern browser available), but has potential to become a serious contender.
      • SerenityOS: Relatively recent project (started 2018) and more of a hobbyist OS, mostly developed by Andreas Kling who however stepped down as the lead programmer in June 2024. Currently still to immature to be used as a daily driver, mostly just runs as a virtual machine at this stage. Interestingly, SerenityOS includes its own browser (Ladybird) developed from scratch, which is suprisingly capable of rendering "modern" websites and now seems to eclipse SerenityOS in popularity.
      • ToaruOS: Relatively recent project and more of a hobbyist OS, mostly developed by K. Lange. Currently still to immature to be used as a daily driver, mostly just runs as a virtual machine at this stage. Doesn't have a modern browser.
    • Other
      • A2 (formerly Bluebottle): A descendant of the Oberon operating system. Never found any widespread adoption or "real-world" use.
      • FreeDOS: While some laptops are available with FreeDOS preinstalled, this OS is not suitable for the average user and lacks modern applications. It's mostly a niche OS for running old DOS software.
      • Fuchsia: Currently only used in niche/embedded devices (e.g. Google Nest). Relatively young OS with unclear roadmap for the future. The graphical user interface (Armadillo) has been removed from Fuchsia. Knowing Google's tendency to kill its projects, it wouldn't be surprising if Fuchsia development will be discontinued suddenly.
      • Genode: One of the most promising and quickly progressing alternative operating systems. Can run a modern web browser (Falkon) and supports a variety of hardware, from PC to Pinephone. Sculpt OS is intended to be a distribution for desktop users, however the GUI is not easy to understand for normal users. Once this runs a "normal" desktop I'd like to include it in the comparison.
      • HelenOS: Research OS from Charles University Prague with no known production use.
      • KolibriOS: Hobby OS without any known production use
      • MenuetOS: Hobby OS without any known production use
      • ReactOS: Attempts to be a free clone of Windows with full binary compatibility, but is decades behind and unlikely to ever catch-up. Currently not stable enough to be used as a daily driver.
      • Nokia Series 30+: OS for modern Nokia "dumbphones".
      • Visopsys: Another hobbyist OS. No modern browser, no real use case.

Appendix I: Can I try out these operating systems in a virtual machine?

The below assumes that you're running Qemu, Virtualbox, or VMWare Workstation Player on a x86_64 (Intel/AMD) PC. Colour guide: possible emulation impossible need to pay

  • Windows: Supported by Qemu/KVM, Qemu/NVMM, Virtualbox, VMware, Hyper-V and many other popular virtual machine hypervisors. Theoretically you need a license key, but an unregistered Windows will still run with minor restrictions (also there's tons of resources on Windows piracy...).
  • macOS: Supported by Qemu/KVM and Virtualbox. Apple does not allow macOS to run on non-Apple hardware, not even in a virtual machine. Nevertheless, there are plenty of guides online how to install a virtual macOS. The future looks bleaker however: Apple will drop support for x86 in 2028, and then you'd be forced to do emulation rather than virtualization on your PC. macOS 26 "Tahoe", to be released in 09/2025, will be the last version of macOS supporting x86 CPUs and will receive security updates until late 2028.
  • iOS/iPadOS: Does not run on x86 hardware and I'm not aware of any emulator with full iOS support. Obviously, Apple would try to shut it down anyway. There is an iOS simulator that is part of XCode (Apple's IDE), however XCode itself is only available for macOS. So if you don't have a Mac, the only way to kind of run iOS would be to install macOS in a VM (see above), then install XCode, and then start the included iOS Simulator.
  • Android: There used to be a project called Android-x86 which made it possible to run "vanilla" Android on a PC (in a virtual machine or on bare metal), but the project has been discontinued. A alternative could be Bliss OS, which is an Android-based operating system for PCs that is in active development, although still lagging behind upstream. It should run on Qemu and Virtualbox.
  • Chrome OS: The "real" Chrome OS that comes preinstalled on Chromebooks is not available for download, however there is Chrome OS Flex, which can be downloaded but has some major restrictions such as no Linux or Android app support. Chrome OS Flex appears to run only on on Qemu.
  • GNU/Linux: Supported by Qemu/KVM, Qemu/NVMM, Virtualbox, VMware, bhyve, Hyper-V, Xen, and many other popular virtual machine hypervisors (even obscure ones such as 9front's vmx or OpenBSD's vmm).
  • FreeBSD: Supported by Qemu/KVM, Qemu/NVMM, Virtualbox, VMware, bhyve, Hyper-V, and Xen.
  • NetBSD: Supported by Qemu/KVM, Qemu/NVMM, Virtualbox, and Xen.
  • OpenBSD: Supported by Qemu/KVM, Qemu/NVMM, Virtualbox, bhyve and Xen.
  • DragonFly BSD: Supported by Qemu/KVM and also runs in Virtualbox using the FreeBSD profile.
  • Illumos: Supported by Qemu/KVM and Virtualbox.
  • Haiku: Supported by Qemu/KVM and I know from my own experience that it also works on Virtualbox.
  • ArcaOS: $$$ Supported by Virtualbox and should also run on VMware and Qemu (though apparently incompatible with KVM as a hypervisor). Please note that you need to pay for an ArcaOS license. Each image is "watermarked" with the license key and due to that it appears that there is virtually no piracy.
  • RISC OS: RISC OS does not run on x86 CPUs so you need an emulator such as Qemu or - specifically for RISC OS - RCPemu. Obviously, emulation is slower than mere virtualization, and setting everything up in Qemu is more complicated than just spinning up a new VM in Virtualbox.
  • AmigaOS: $$$ AmigaOS does not run on x86 CPUs so you need an emulator such as Qemu or - specifically for AmigaOS - UAE. Obviously, emulation is slower than mere virtualization, and setting everything up in Qemu is more complicated than just spinning up a new VM in Virtualbox. Also don't forget that you need to pay for AmigaOS (I've also not seen any pirated ISOs).
  • MorphOS: $$$ MorphOS does not run on x86 CPUs so you need an emulator such as Qemu. I've managed to make it run using these instructions. Obviously, emulation is slower than mere virtualization, and setting everything up in Qemu is more complicated than just spinning up a new VM in Virtualbox. MorphOS is paid, but you can download the image for free. If you don't have a license, the system will slow to a crawl after 30 minutes until you reboot, so it's still good enough for playing around. Unfortunately, the license key is tied to the hardware and because of that you won't be able to purchase a license key for a virtual machine (because it could be copied and shared with others). So you'll be limited to the 30 minutes.
  • AROS: Should work on Qemu, Virtualbox, VMware although not officially supported by either.
  • OpenVMS: According to the developer, it should work on Qemu, Virtualbox, and VMWare. OpenVMS is paid software targeting enterprise customers but there is a free community license. Some guidance on installing it in a VM: [1], [2], [3]. Update: Since 2024, the hobbyist license doesn't allow you to get the installation media anymore. Instead: "A vmdk of a system disk with OpenVMS V9.2-2 and compilers installed and licensed will be provided, along with instructions to create a virtual machine and the SYSTEM password. The license installed on that system will be valid for one year, at which point we will provide a new package." [4] So in other words, you'll have to set everything up anew after a year unless you pay for the commercial license.
